# V1 设计（目标导向）

## 目标
1. 尽可能抓到“可用咨询”（原帖、回复、评论线索、外链、图片）。
2. 形成可消费产物：
- 日报：快速知晓当天变化。
- 帖子流：快速浏览与筛选。
- 专题对比：和现有想法/方案对照验证。

## 采集范围（V1）
1. 帖子基础：作者、时间、文本、链接、互动指标。
2. 关系信息：是否回复、父帖ID、会话ID。
3. 资产信息：外链、图片链接、本地图片路径。
4. 主题分类：工具分享、使用体验、新闻/论文。

## 数据处理（V1）
1. 标准化：统一字段、时间格式、作者字段。
2. 去重：按 `tweet_id`（无ID时使用文本+时间哈希）去重。
3. 分类：规则引擎（关键词 + 域名）。
4. 比较：对“现有方案”做支持/冲突证据匹配。

## 输出物（V1）
1. `daily_brief_v1.md`：日报（关键变化 + Top 信号）。
2. `post_feed_v1.csv`：统一帖子流（含分类）。
3. `idea_compare_v1.csv`：方案对比矩阵（支持/冲突/证据链接）。
4. `hub_v1.html`：可视化页面（日报/帖子/对比三屏）。

## 浏览方式（V1）
1. 先看日报：5 分钟掌握。
2. 再看帖子流：按作者、分类、关键词筛选。
3. 最后看对比：针对候选方案做证据决策。
